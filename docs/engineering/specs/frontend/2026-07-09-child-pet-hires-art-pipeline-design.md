# 孩子端宠物高清美术产线设计（AI 生成 · 火龙 5 阶试点）

- 日期：2026-07-09
- 状态：设计确认完成（brainstorm 通过，待写实现计划）
- 范围：把 `frontend/child-web-prototype/child-homepage.html` 里孩子端宠物的**矢量方向稿精灵**升级为 **AI 生成的高清位图**；本轮**只做火龙 5 阶试点**，验证后再铺恐龙 / 光之英雄
- 承接：`2026-07-09-child-homepage-pet-redesign-design.md` §7 / §11.4（"英雄卡通 SVG 方向稿 → 上线前可换高清美术，不动结构"）

## 1. 背景与目标

宠物重设已落地并合并进 main：三物种（火龙/恐龙/光之英雄）× 5 阶，用内联 SVG `<symbol>` 精灵渲染，属"英雄卡通方向稿"精度。本产线把这层"皮"换成**高清位图美术**，观感更接近成品游戏，同时**不改动** SPECIES 数据、spriteId、渲染/进化/图鉴/选宠物等逻辑。

**关键约束（诚实前提）**：Claude 无法直接生成位图/绘画类美术。本产线的分工是：**Claude 出「参考图 + 提示词 + 一致性策略 + 接线代码」，创始人用国内 AI 绘图工具（即梦/通义/文心等）跑图，Claude 再把成品接回原型。**

## 2. 核心决策

| 维度 | 决策 |
|---|---|
| 产出路线 | **AI 生成位图**（非委托、非纯矢量精修） |
| 绘图工具 | **国内工具**（即梦/通义/文心等），中文提示词，支持图生图/参考图 |
| 一致性地基 | **用现有 15 张 SVG 精灵当"结构参考图"**（本轮先用火龙 5 张）——剪影/姿态/构图/配色身份已确认，喂参考图锁结构、只换质感 |
| 分批 | **先火龙 5 阶试点**，验证质感 + "同一只龙长大"一致性，再铺恐龙/光之英雄 |
| 存放 | 外部 `frontend/child-web-prototype/assets/pets/`（不做 base64 内联） |
| 格式 | 透明 PNG，统一 **1024×1024**，命名 `sp-<species>-<1..5>.png` |
| 集成 | **位图优先、缺图回退矢量**：有 PNG 用 `<image>`，无则用现有 SVG `<symbol>` |

## 3. 产线五块

### 3.1 参考图导出（Claude 做）
用 Playwright 把火龙 5 个 SVG 精灵（`sp-dragon-1..5`）各渲染到**透明背景**、居中、统一画布（1024×1024，四周留一致边距）导出为 PNG：`ref-dragon-1.png`…`ref-dragon-5.png`。这 5 张是喂给即梦的**结构参考图**，锁定每阶剪影/姿态/构图/配色。

### 3.2 美术设定 + 逐张中文提示词（Claude 做）
**统一风格锚点**（附加到每一张提示词，保证 5 阶同风格）：
- 风格：英雄卡通 / 暴雪卡通游戏角色感（《皇室战争》《炉石》向），柔和 cel-shading + 边缘轮光 + 柔和高光
- 火龙身份：暖橙红主体、奶黄腹甲、发光胸核、尾焰；每阶保留身份锚点
- 出图规范：**透明背景**、单个角色居中、正面略 3/4、无地台/无阴影、儿童向明亮不暗黑、无文字/无边框
- 负向提示：写实恐怖、暗黑、多角色、地面阴影、文字水印、粗糙线条

**5 段分阶描述**（对应已确认形态）：
1. `sp-dragon-1` 龙蛋：暖色蛋 + 发光裂纹，神秘感
2. `sp-dragon-2` 破壳萌龙：超大圆头巨眼奶龙、蛋壳、尾焰，纯萌
3. `sp-dragon-3` 成长幼龙：站立、小角小背刺、小吻，仍不会飞
4. `sp-dragon-4` 展翼幼龙：长出翅膀、口鼻/眉骨、脖子，眼神变利
5. `sp-dragon-5` 喷火成龙：长吻龙头 + 大弯角 + 獠牙 + 双翼 + 喷火，英雄战龙

**一致性用法说明**（写给创始人的操作指引）：
- 每张：即梦「图生图/参考图」，参考图 = 对应阶的 `ref-dragon-N.png`，参考强度调到「保结构、换质感」区间（结构锁住、风格由提示词决定）
- 相邻阶强化：出完第 N 阶成品后，把该成品当**二次参考**再生成第 N+1 阶，强化"同一只龙长大"的连续性
- 固定风格种子 / 参考同一张成品做风格基准，减少漂移

### 3.3 交付规范
创始人交付：透明 PNG × 5，统一 1024×1024，居中同比例，命名 `sp-dragon-1.png`…`sp-dragon-5.png`，放入 `frontend/child-web-prototype/assets/pets/`。（Claude 提供一个校验脚本：尺寸/透明/命名/大致居中，防止接回时错位。）

### 3.4 接回原型（Claude 做）
在渲染层做**位图优先、缺图回退矢量**：
- `SPECIES.dragon.stages[N]` 增加可选 `art`（位图路径）或用约定路径 `assets/pets/<spriteId>.png`；
- `renderPetStage` / `renderEvolutionRail` / 选宠物卡片：若对应位图存在 → 用 `<image href="assets/pets/<spriteId>.png">`（或 `<img>`/CSS 背景）显示高清图；否则回退现有 `<use href="#<spriteId>">` 矢量。
- 位图沿用同一套定位/尺寸/`viewBox` 框架（延续 pet-redesign 的统一包络与图鉴同地平线布局），保证接回后主舞台/图鉴/选宠物三处位置一致。
- SPECIES 数据、spriteId、进化演出、循环逻辑**不改**。

### 3.5 试点验收
- 主舞台 / 图鉴 / 选宠物三处均显示高清火龙；
- 5 阶质感统一、"同一只龙长大"连续可辨；
- 透明边干净无白边/无锯齿；移动端（~430px）清晰不糊；
- 缺图回退矢量正常（把某张 PNG 移走仍不崩、显示矢量）；
- 满意后照同一产线产恐龙 / 光之英雄（各自身份色 + 参考图 + 分阶提示词）。

## 4. 后续扩展（试点通过后）
- 恐龙（绿系，招牌背板尖刺/咆哮）与光之英雄（蓝银光，招牌铠甲光刃/光束）照同一模板：导参考图 → 分阶提示词 → 跑图 → 接回。
- 光之英雄提示词须**原创**、不含奥特曼官方形象/名称/配色比例（延续 pet-redesign §3.3 约束）。
- 进化演出的光爆/粒子仍由 CSS/SVG 覆盖层实现，与位图角色叠加即可，无需为动画单独产图。

## 5. 非目标 / 风险
- 不改 SPECIES/spriteId/进化逻辑/交互；不做骨骼动画或多帧序列（角色是静态高清图，动感靠现有 CSS 覆盖层）。
- **AI 一致性风险**：15 张（先 5 张）跨阶一致性是最大不确定项——本设计用"SVG 参考图 + 相邻阶二次参考 + 固定风格基准"缓解，且先试点验证再铺开。
- **产图依赖创始人**：Claude 不产图；若创始人无法产出满意位图，可回退继续用现有矢量（缺图回退机制保证不阻断）。
- 体积：15 张 1024 PNG 体积可控（数百 KB~1MB 量级），置于 `assets/`。
