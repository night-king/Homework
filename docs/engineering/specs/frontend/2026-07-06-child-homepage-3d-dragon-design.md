# 孩子端首页 3D 圣龙舞台设计稿

- 日期：2026-07-06
- 状态：设计确认完成，进入单文件原型实现
- 范围：仅覆盖 `frontend/child-web-prototype/child-homepage.html` 的首页主舞台圣龙形象与舞台交互

## 目标

把孩子端首页的主角从“平面 CSS 小龙”升级成“强烈 3D 观感、可拖动旋转、会主动回应孩子”的圣龙伙伴舞台。

本次改造重点不是补更多后台逻辑，而是把首页最关键的情绪锚点做强：

- 一眼看出是 3D 圣龙，而不是平面拼贴
- 鼠标拖动即可旋转观察，增强“它真的在台上”的感觉
- 5 阶进化在轮廓层面明显变化，不再只是小配件升级
- 喂食、打盹、说话三类动作可以直接在首页发生

## 体验主线

首页主循环保持不变：

1. 完成委托
2. 获得补给
3. 喂给圣龙
4. 圣龙做出明显反应
5. 成长 / 可进化 / 解锁新形态

在这条主循环之外，增加两类轻情绪互动：

- `说话`：孩子点一下，圣龙会回一句鼓励或撒娇
- `打盹`：孩子可主动让伙伴短暂休息，看到更柔和的一面

## 视觉方向

### 总体气质

- 玩具感 / 手办感 / 儿童幻想冒险感
- 低多边形卡通圣龙，不做写实鳞片怪兽
- 保留暖色主体，但允许五行核心改变胸口、尾焰、翼纹、冠饰的能量颜色

### 舞台氛围

- 保留原有 `aura / disc / energy ring` 作为舞台能量底座
- 圣龙本体改为 `canvas` 渲染的单文件 3D 伪场景
- 继续使用 HTML 浮层承接：
  - 状态标签
  - 反馈气泡
  - 进化卡片
  - 五行切换器
  - 互动按钮

## 5 阶形态设计

### 1 阶｜焰团幼龙

- 头身比最大
- 身体短圆，腿短，尾焰最夸张
- 没有完整翅膀和王冠，整体像“火团长出小龙骨架”
- 关键词：可爱、需要照顾

### 2 阶｜迅爪少龙

- 躯干明显拉长
- 腿更高，前倾，冲刺感更强
- 长出短角和背脊刺
- 仍然不飞，但已经像闯关伙伴
- 关键词：灵活、少年感

### 3 阶｜展翼飞龙

- 第一次完整展开大翼
- 头身比下降，脖子拉开，胸口抬起
- 从“地面跑龙”切换为“天空伙伴”
- 关键词：第一次真正的大进化

### 4 阶｜重甲守护龙

- 胸腔和肩线显著变宽
- 护甲、护臂、胸核都变厚重
- 翅膀不再只强调轻盈，而强调守护者体格
- 关键词：可靠、能挡在主人前面

### 5 阶｜圣冠龙王

- 长颈、长翼、长尾，整体更高更庄严
- 王冠完整展开，并与龙角连成统治者轮廓
- 圣甲和胸核升级为“加冕形态”
- 关键词：荣耀、守护、终局感

## 交互定义

### 拖动旋转

- 默认展示 3/4 英雄角度
- 鼠标按住可左右拖动旋转
- 上下拖动只允许小范围俯仰
- 松手后缓慢回弹到默认角度
- 不做自由轨道相机，避免失控

### 常驻生命感

- 呼吸起伏
- 眨眼
- 尾巴摆动
- 尾焰 / 胸核轻微脉冲
- 3 阶以上翅膀存在低频展开呼吸

### 动作反馈

- `喂食`：抬头接食、身体发亮、尾焰增强
- `打盹`：身体下沉、眼睛闭合、冒出小气泡或 `Zzz`
- `说话`：头部点动、冒泡说话
- `摸摸`：点击龙身时有轻微蹭头反馈，可作为隐藏情绪互动

## 状态映射

继续沿用首页原有状态机，并映射为 3D 动作气质：

- `rest`：身体更松，眼神更软，动作频率变慢
- `idle`：待命站姿，轻呼吸
- `ready`：更有精神，尾巴更活跃
- `hungry`：明显期待，被喂之前会更躁动
- `complete`：更自豪，翅膀与胸核庆祝性增强
- `evolve`：整体光晕增强，进入高能待进化姿态

## 技术方案

- 维持 `child-homepage.html` 单文件原型
- 使用 `canvas + 原生 JavaScript` 构建轻量 3D 伪渲染器
- 不引入资产流水线，不拆分模型文件，不要求真实骨骼动画
- 通过几何体组合、投影、明暗、层级动画形成“低多边形 3D 手办感”

### 为什么不用平面 CSS 继续堆

当前纯 CSS 方案虽然能做造型，但不适合以下目标：

- 明显的拖动旋转
- 轮廓级进化差异
- 更自然的空间感与遮挡关系
- 后续扩展多个动作状态

因此首页主舞台改成 `canvas` 更合适，但仍然保持单文件、轻依赖、可快速迭代。

## 保留与不改动

### 保留

- 顶部状态条
- `Day` 切换
- 今日委托区
- 补给 / 喂食成长循环
- 图鉴弹窗与阶段数据结构
- 五行切换入口与进化卡片布局

### 暂不做

- 真正的 3D 资源文件加载
- 真实骨骼系统
- 完整战斗演出
- 移动端复杂手势编排
- 多页联动

## 实现验收标准

满足以下条件即可认为本轮原型达标：

- 首页主龙明显具备 3D 观感
- 拖动旋转手感成立，且会自动回到英雄角度
- 5 阶轮廓差异一眼可辨
- `喂食 / 打盹 / 说话` 都能直接触发动作反馈
- 现有任务完成、成长、进化、五行切换逻辑不被破坏
