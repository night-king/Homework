# 孩子端首页宠物重设 · 三角色可选 + 5 阶进化设计稿

- 日期：2026-07-09
- 状态：设计确认完成（brainstorm 通过，待写实现计划）
- 范围：`frontend/child-web-prototype/child-homepage.html` 的首页主舞台宠物形象、开局选宠物、5 阶进化路线与进化演出
- 取代：`2026-07-05-child-homepage-sacred-dragon-design.md` 与 `2026-07-06-child-homepage-3d-dragon-design.md`（原"单一圣龙 + 五行 + 伪 3D 舞台"方向），这两份归档

## 1. 背景与动机

现有首页宠物（圣龙 / 伪 3D canvas 低多边形）存在两个创始人明确不满意的点：

1. **造型画风太粗糙**：手搓 canvas / CSS 几何，达不到当下小学生审美，勾不起兴趣。
2. **进化路线气质不对**：原"一出场就是会喷火的圣龙 → 战力升级 → 加冕龙王"是战力叙事；创始人想要"亲手养大一条生命"的成长叙事（蛋 → 孵化 → 长大 → 长本事）。

本次重设解决这两点，并把愿景文档里早已确认、但一直没落地的"**开局自选 3 种宠物**"机制真正做出来。

## 2. 核心决策总览

| 维度 | 决策 |
|---|---|
| 宠物数量 | **3 只完全独立的角色**，孩子开局自选 1 只（自主感 / 专属感） |
| 三角色 | 🔥 火龙（红橙）· 🦖 恐龙（绿）· ⚡ 光之英雄（蓝银光，原创，避开奥特曼版权） |
| 统一画风 | **英雄卡通**（《皇室战争》《炉石》/ 暴雪卡通那类游戏角色感）：厚重有魄力、发光眼神、战斗站姿，但配色明亮不吓人，适配 6–8 岁 |
| 进化节奏 | 每只都是 **5 阶**，共享同一条节奏：神秘容器 → 萌态揭晓 → 成长 → 招牌觉醒 → 终极大招 |
| 情感弧线 | **从萌到酷**：小时候萌（蛋 / 大眼奶崽），越长越酷（终极大招形态） |
| 进化演出 | 每次跨阶都有**进化过渡动画**（蓄力→光爆→揭晓→庆祝 4 拍）+ 各阶专属揭晓 |
| 五行系统 | **砍掉**（原"同一条龙解锁五行器官"被"3 独立角色"取代，简化为每只单一身份），未来若要再议 |

## 3. 三角色 5 阶进化路线

三只共享节奏与玩法机制，但各有身份配色与专属招牌。招牌能力从"成长期获得"到"终极大招"逐级揭晓，不再是一出场就有。

### 3.1 🔥 火龙（红橙系）

| 阶 | 形态 | 轮廓 / 头部要点 | 揭晓时刻 ★ |
|---|---|---|---|
| 1 | 龙蛋 | 暖色蛋、发光裂纹随进度变多 | — |
| 2 | 破壳萌龙 | 超大圆头 + 巨眼、无吻部，尾焰 | ★ 裂壳光爆→奶龙探头 |
| 3 | 成长幼龙 | 站立、冒小吻 + 小角 + 背刺，仍不会飞 | — |
| 4 | 展翼幼龙 | 头拉长 + 口鼻 + 眉骨 + 脖子，背后**翅膀第一次展开** | ★ 展翅（地面→天空） |
| 5 | 喷火成龙 | 侧头长吻龙头 + 大弯角 + 獠牙 + 凶眼，**首次喷火** | ★ 首次喷火 |

- 招牌成长：翅膀 → 喷火。身份锚点：尾焰。

### 3.2 🦖 恐龙（绿系）

| 阶 | 形态 | 轮廓 / 头部要点 | 揭晓时刻 ★ |
|---|---|---|---|
| 1 | 恐龙蛋 | 绿壳带斑点、发光裂纹，比龙蛋更圆 | — |
| 2 | 破壳幼崽 | 大圆头巨眼小奶崽，头顶小冠芽 | ★ 破壳探头 |
| 3 | 小恐龙 | 双足站立、拖长尾、背上冒小背板 | — |
| 4 | 甲刺觉醒 | **背板尖刺 + 头冠 frill + 鼻角**长出，身体变壮 | ★ 甲刺觉醒 |
| 5 | 霸王战龙 | 大 T-Rex：血盆大口 + 獠牙，**首次震地咆哮**（青色声波） | ★ 首次咆哮 |

- 招牌成长：背板尖刺 → 震地咆哮（非喷火）。

### 3.3 ⚡ 光之英雄（蓝银光系 · 原创）

人形英雄，"年龄感"靠**头身比**（Q 版矮胖 → 巨神高挑）表达，就像龙靠头型。第 1 阶不是蛋，而是"从光中诞生"的**光之核**（能量水晶），一样保留神秘揭晓。

| 阶 | 形态 | 轮廓 / 比例要点 | 揭晓时刻 ★ |
|---|---|---|---|
| 1 | 光之核 | 发光能量水晶 + 光裂纹（替代蛋） | — |
| 2 | 觉醒小英雄 | Q 版矮胖（约 2 头身）、大头盔、发光眼、胸口光核 | ★ 从光中觉醒 |
| 3 | 成长斗士 | 约 3 头身、身形舒展、出现胸甲 | — |
| 4 | 铠甲光刃 | 约 4 头身、肩甲 / 护甲全上 + 手持**光刃** | ★ 铠甲 + 光刃 |
| 5 | 光之巨神 | 约 5–6 头身、金冠 + 全甲 + 发光气场 + **光束**大招 | ★ 首次光束 |

- 招牌成长：铠甲光刃 → 光束。**版权**：借"光之巨人 / 铠甲英雄"美学做**原创**，不使用奥特曼官方形象、名称、配色比例。

### 3.4 三角色对照

| 角色 | 配色 | 第 1 阶容器 | 招牌成长 | 终极大招 |
|---|---|---|---|---|
| 🔥 火龙 | 红橙 | 龙蛋 | 翅膀 | 喷火 |
| 🦖 恐龙 | 绿 | 恐龙蛋 | 背板尖刺 | 震地咆哮 |
| ⚡ 光之英雄 | 蓝银光 | 光之核 | 铠甲光刃 | 光束 |

## 4. 造型规范（三只共用）

- **画风强度**：英雄卡通档（够酷 / 够游戏感，但明亮圆润不暗黑，6 岁不怕）。渐变体积、发光眼神、舞台底盘（能量圆 / 光盘）、边缘轮光（rim light）、发光核心。
- **个头递增**：5 阶站在**同一条地平线**上，按真实比例递增——从"小小的容器"到"顶天的终极形态"，成长落差一眼可见。（这是首页宠物舞台里最重要的一条阅读规则。）
- **头部 / 比例差异**：每阶必须让"头部（生物）或头身比（英雄）"发生明显变化，这是年龄感最强的信号。生物：圆奶头 → 长吻成兽头；英雄：大头 Q 版 → 高挑巨神。
- **每阶只强化 1–2 个大结构**，保证"同一只一路长大"的连续性，同时每阶有新鲜感。
- **身份锚点**：火龙=尾焰，恐龙=背板+绿，英雄=胸口光核+蓝银。全程保留，帮助识别"同一只"。

## 5. 进化过渡动画

每次跨阶（成长值越过阶段阈值）触发一段**进化演出**，是"神秘揭晓"的重头戏，接现有里程碑鼓励机制。

**通用 4 拍：**

1. **蓄力**：当前形态发光、微微颤抖，舞台能量环收拢。
2. **光爆**：一道白光盖住画面。
3. **揭晓**：光散开，新形态放大登场 + 放射光线。
4. **庆祝**：星星迸出 + 鼓励语气泡。

**关键实现技巧**：不做逐帧几何变形（成本高、脆弱）；用**"光爆盖住那一瞬间完成换装"**——白光最亮时把旧形态换成新形态，光一散即新形态。视觉上是真变身，实现简单稳定。

**每阶专属揭晓**（在通用演出基础上叠加）：破壳探头 / 展翅 / 首次喷火 / 甲刺撑开 / 首次咆哮 / 光之核绽放 / 抽出光刃 / 首次光束。

## 6. 开局选宠物（Selection UX）

- 孩子第一次进入游戏端时，进入"选择伙伴"界面：三只角色**以第 1 阶容器 + 一句气质文案**并排展示，孩子点选 1 只。
- 选定后即成为该孩子专属宠物；哥哥 / 弟弟可各选不同角色，排行榜也更好区分。
- 对应愿景文档 §5.5 "开局自选宠物（v1 提供 3 种）"。选择行为本身承担"专属感"，因此画风不再作为可选项（三角色统一英雄卡通）。

## 7. 技术方案与边界

- **维持单文件原型**：`child-homepage.html`，用 **SVG + CSS/JS 动画**实现三角色 5 阶造型与进化演出；替换原 canvas 伪 3D 渲染器。
- **英雄卡通用矢量实现**：渐变、发光滤镜、轮光、粒子/光爆用 SVG + CSS 完成，单文件、轻依赖、可快速迭代。
- **美术精修留口子**：本轮 SVG 是"造型方向稿"，用来敲定轮廓 / 比例 / 进化节奏与配色；上线前这套英雄卡通质感可再用高清美术（AI 生成 / 委托）精修替换，不影响结构与逻辑。
- **暂不做**：真实 3D 资源 / 骨骼动画 / 自由旋转相机（本轮 2D 插画方向已放弃 3D 旋转）/ 复杂手势编排 / 多页联动。
- **保留不改**：现有首页壳（顶部状态条、Day 切换、今日委托区、补给→喂食成长循环、图鉴弹窗、互动按钮 喂食/打盹/说话/摸摸），本稿只替换"宠物舞台的形象系统 + 进化路线 + 进化演出 + 选宠物入口"。

## 8. 与后端 / 数据的映射

- 本轮是**前端原型**改造，不需要后端改动即可推进。
- 未来接后端时天然对齐现有领域模型：`Pet.Species`（三角色 = 三 species）、`Pet.Stage`（5 阶）、`Pet.GrowthPoints`（单调只增，驱动阶段阈值）。阶段阈值按愿景 §5.5 做数据驱动（4–5 阶，可调设置项），本设计取 5 阶。
- 三只共享同一套成长值 / 里程碑机制，只是形态与招牌不同 → 后端逻辑保持统一。

## 9. 范围与实现分期建议

- 本设计**覆盖三只角色全部 5 阶**。
- 实现可分期落地（降低一次改坏风险），建议顺序：**先火龙整条线做透（含进化演出）→ 再恐龙 → 再光之英雄**，最后补选宠物入口联动。每只内部再按"1 阶容器 → 终极 → 中间过渡"推进。
- 这只是**实现排期建议**，不改变"三只都是一等公民"的设计结论。

## 10. 验收标准

- 首页宠物明显是"精致英雄卡通"，肉眼即感到比旧版粗糙造型有质的提升。
- 三角色身份清晰、一眼区分（红火龙 / 绿恐龙 / 蓝光英雄）。
- 每只 5 阶**个头递增可辨、头部/比例差异明显**。
- 每次跨阶有**进化演出动画**，且各阶有专属揭晓。
- 开局可从三只中自选 1 只，成为该孩子专属宠物。
- 现有首页任务完成 / 成长 / 里程碑 / 互动逻辑不被破坏。

## 11. 待创始人复审确认的点

1. 五行系统确认**砍掉**（本稿默认已砍）。
2. 光之英雄第 1 阶用"光之核"而非"蛋"，是否认可（三只容器不完全同形，但都保留神秘揭晓）。
3. 实现分期先做火龙是否认可（设计不变，仅排期）。
4. "英雄卡通 SVG 方向稿 → 上线前高清美术精修"这条路径是否认可，还是希望某一步就直接接入生成 / 委托美术。
