# Docs

`docs/` 只放文档，不放可直接运行的页面或原型。

## 分类

- `product/`：产品愿景与 backlog
- `engineering/specs/`：当前有效的工程设计文档
- `engineering/plans/`：正在执行或即将执行的实施计划
- `engineering/runbooks/`：运行手册 / 操作手册
- `marketing/`：官网与营销内容
- `archive/`：历史资料、已完成计划、过时实现文档

## 规则

- 可运行前端原型应放在 `frontend/`，不要再放进 `docs/`
- 新文档按“用途”归档，不按工具来源归档
- 已完成的实施计划优先移到 `archive/`
- 如果文档主要用于历史追溯，请移动到 `archive/`
