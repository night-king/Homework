# 儿童暑假作业激励游戏 · 设计方案

- **日期**：2026-07-04
- **状态**：设计已确认（评审通过 · 可进入实现计划）
- **技术栈**：ABP Framework 10+（.NET）· MySQL · ABP MVC 单体应用
- **形态**：全栈 Web 应用（家长后台 + 孩子游戏端），移动端优先，PWA 可安装

---

## 1. 背景与目标

家里两个孩子（一个即将上一年级 ≈6 岁，一个即将上三年级 ≈8 岁），暑假老师布置了每日家庭作业，但孩子积极性不高、难以自主完成每日学习任务。

**目标**：做一款有趣的网页游戏，用"养宠物 + 良性竞争"把"完成每日作业"变成孩子每天主动想做的事。核心诉求：

1. 每天按 schedule 完成学习任务；每完成一个就能给自己的小宠物喂食。
2. 宠物要吃饱，就必须完成当天所有任务。
3. 每完成若干任务达成里程碑，宠物说一句鼓励的话。
4. 有日榜和总榜，让两个孩子形成竞争、互相鼓励。
5. 趣味性足、契合小学生审美但不过分花哨。
6. 数据要能长期存储（长期项目，非一次性）。

**非目标（本期不做）**：面向多用户/多家庭的 SaaS、社交网络、内购、复杂反作弊、AI 批改作业内容。本项目服务于"这一个家庭的两个孩子"，但架构上不排斥未来扩展。

## 2. 设计原则（儿童心理学）

贯穿全案的几条底层原则，用来指导所有取舍：

- **自我决定理论（自主 / 胜任 / 关联）**：给孩子自主感（自评打卡、自选宠物）、胜任感（可达成、公平的目标）、关联感（合作大目标、家长鼓励语）。
- **成长型思维（Dweck）**：一切鼓励语只夸"努力 / 坚持 / 策略"，绝不夸"聪明 / 天赋 / 分数高"，避免脆弱自尊。
- **精熟目标 > 表现目标**：竞争的是"有没有完成自己的计划"，而非"谁比谁强"，弱者永远有翻盘机会。
- **公平竞争才有激励**：6 岁与 8 岁作业量天生不同，必须用"完成度"而非"绝对数量"作为竞争基准，否则低龄孩子习得性无助、放弃。
- **正向 / 低压力**：宠物机制以爱与期待驱动，不用恐惧与愧疚驱动；即时反馈、即时成就感。
- **即时反馈**：完成即喂食、即得星、即鼓励，缩短"付出→回报"的心理距离。

## 3. 使用者与角色

| 角色 | 说明 | 权限 |
|------|------|------|
| 家长（管理员） | 布置任务、复核打卡、设家庭大目标与奖励、（后期）自定义鼓励语 | ABP 管理员，后台全权限 |
| 孩子 ×2（哥哥 / 弟弟） | 玩游戏端：看今日任务、打卡喂宠物、看排行榜 | "Child" 角色，仅能读写**自己**的数据；只读排行榜里对方的公开分数 |

- 使用 **ABP Identity** 建账号：1 个家长（Admin），2 个孩子（Child 角色）。
- 权限用 ABP Authorization：Child 只能访问自己 `ChildId` 的任务/宠物/得分；无法修改对方数据、无法进入后台。

## 4. 运营模型

### 4.1 任务日程：每周模板 + 可微调
- 每个孩子各有一套**每周任务模板**（周一~周日，每天一组任务，如"语文朗读 20 分钟 / 数学口算一页 / 英语单词 10 个 / 课外阅读 15 分钟"）。
- 系统每天按当天星期**生成当日任务**（`DailyTask`）——采用**惰性生成 + 过去日补档**（见 §7.7），保证账本无缺口、不会因定时任务漏跑而缺档。
- 家长可对"某一天的当日任务"临时增删改，不影响模板。
- 两个孩子年级不同 → 模板/任务各自独立。

### 4.2 完成确认：孩子自评 + 家长可复核
- 孩子在游戏端自己给任务打勾，**立即**喂宠物、得星、触发鼓励（保护自主感与即时成就感）。
- 所有打卡带时间戳，进 ABP 审计。
- 家长后台可看到完成记录，**事后**可"撤销 / 修正"异常打卡（信任为主、轻度约束，不做打断式审核）。
- **撤销 / 修正的连锁重算**：`DailyScore` 是唯一的记分账本（source of truth）。家长改动某天打卡后，系统重算**当天** `DailyScore`（完成数 / 星星 / 是否吃饱），其余派生量自动跟随：
  - 日榜、总榜累计分、家庭大目标进度，均由 `DailyScore` **聚合求和实时得出**（不额外持久化可变汇总值）。
  - 连击由 `DailyScore` 历史**重新推导**。
  - **唯一例外**：宠物 `GrowthPoints` 单调只增，**不因撤销而回退**（见 §5.5）。
  - 低数据量（两娃一个暑假）下聚合计算成本可忽略，从而避免增量维护的级联 bug。

### 4.3 登录：设备记住身份，自动登录
- 每台设备第一次选好"我是谁"，之后打开自动进入该孩子账号（长效持久化登录，绑定该浏览器）。
- 可选 **4 位数字 PIN**，防止兄弟互换设备捣乱 / 切换身份时校验；PIN 生命周期见 §7.2。
- 家长后台走标准 ABP 登录。

## 5. 激励模型

### 5.1 每日星星（竞争基准，公平核心）
- 设当天该孩子计划任务数为 `N`，已完成 `C`，完成比例 `r = C / N`。
- **每日星星 `stars = ceil(r × 5)`**，封顶 5★（`N>0` 且 `C=0 → 0★`；全部完成 `C=N → 5★=吃饱`）。
- 两个孩子只要各自完成自己的计划，都能拿满 5★、都是赢家；满星平局是常态，要被正面庆祝（"你俩今天都满星！"）。
- **无任务日（`N=0`：休息日 / 家长清空当天 / 模板尚未建立）**：不计算比例，`stars=0`，该日**不参与日榜名次**（显示"休息日 / 暂无任务"），**既不算"吃饱"也不算断连击**（连击计算中直接跳过此日，见 §5.3）；宠物当天为"没有任务、悠闲"状态（既不喂食也不变蔫）。
- `5`（星星上限）、取整方式等做成 **ABP 设置项**，家长后台可调，无需改代码。

### 5.2 日榜 / 总榜
- **日榜**：比当天星星，**每天 0 点清零重来**，弱者永远有翻盘机会。
- **总榜**：暑假**累计星星**（= 各日 `DailyScore.Stars` 求和）+ 🔥**连续打卡天数**。
- 排行榜同屏展示两个孩子；文案强调"完成自己的计划"，弱化"碾压对方"。

### 5.3 连续打卡（Streak）
- **默认定义**：连续多少天**有任务且达到"吃饱"（100% 完成）**的天数；**无任务日（`N=0`）直接跳过**——既不中断也不延长（起"桥接"作用）。
- **计算来源**：由 `DailyScore` 完整账本历史**推导**得出（非独立可变状态），故家长事后改打卡后连击自动重算（§4.2）。`StreakInfo` 仅是可重算的**投影 / 缓存**，永不作为真相源，不会与账本分歧。
- **缺档日判定（关键）**：过去某天若"没人打开 App"，惰性生成本不会为它建行——但**缺档 ≠ 没发生**，不能被连击忽略。规则：该星期几**模板有任务（`N>0`）却无完成 = 漏做 = 断连击**；模板**本就无任务（`N=0`）= 休息日 = 桥接**。实现上由"访问时结算过去日"把账本补齐（§7.7），再据无缺口账本推导连击。
- `StreakInfo.LastFullDate` 在当天首次达到 100% 时写入；若家长当天**又加了一项任务**导致完成度跌回 <100%，该日不再算"吃饱"，连击相应回退（因为它是从 `DailyScore` 推导的）。
- 断签只是归零重来，不做任何惩罚或负面文案。
- 里程碑：连续 3 / 7 / 14 / 30 天触发特别鼓励与庆祝。
- （v2 考虑"补签 / streak 冻结"以减少长连击断裂的挫败感。）

### 5.4 合作大目标（关联感，降低零和对立）
- 家长设一个 `FamilyGoal`：目标星星数 + 奖励文案（如"一起攒 200★ → 全家看电影 🎬"）+ 时间范围。
- 进度 = 两个孩子在该范围内星星之和（由 `DailyScore` 聚合）；达标 → 全家庆祝 + 展示奖励。
- 可先后设多个大目标。

### 5.5 宠物系统
- 每个孩子**开局自选一只**宠物（自主感、专属感 → 更强责任心）。v1 提供 **3 种**可选。
- **喂养**：每完成一项任务喂一次，饱食度 = 当天完成比例；全做完 = 吃饱 + 当日奖励。
- **成长进化**：按**成长值 `GrowthPoints`** 分阶段成长（蛋 → 幼崽 → … → 进化形态），长期有盼头。
  - `GrowthPoints` 是**单调只增的成长计数器**，定义为"**历史最高的累计已得星星**（high-water mark）"：累计已得星星上升时它跟着上升，**撤销 / 反复勾选取消都不会让它回退或虚增**（与"可增可减的总榜累计分"是两个不同的量，见 §4.2、§6）。这样"成长只涨不掉"的承诺与"打卡可撤销"不矛盾，也不会被反复切换刷高。
  - v1 默认阶梯（可调、数据驱动）：约 4~5 个阶段，按 `GrowthPoints` 阈值提升。
- **温和饥饿 + 安全护栏**（关键）：
  - 宠物当天的**饱食度 / 心情是从"今天的完成度"实时推导**的（不是需要定时重置的存储字段）——因此"**第二天满血复活**"是自动的：新的一天 = 新的空白 `DailyScore` = 宠物"饿了、期待吃饭"，**不依赖任何定时任务**（§7.7）。
  - 漏做任务，宠物只表现为"有点困 / 耷拉着 / 没精神"；**绝不**哭闹、生病、离开、死亡。
  - 放大"重新活力满满"的回归时刻，把牵挂转成温柔动力而非愧疚。

### 5.6 里程碑与鼓励语
- **触发点（默认，阈值可调）**：
  - 当天完成一半（≥50%）、当天全部完成（100%）。
  - 连续打卡 3 / 7 / 14 / 30 天。
  - （每次喂食有轻量宠物反应动画，但不一定弹鼓励语。）
- **鼓励语来源**：
  - 内置一套**分龄、成长型思维**的中文语库（多样、不易重复；夸努力/坚持）。
  - 家长后台可添加自己的话（如"妈妈为你骄傲"），对孩子格外有力量。
- 语库按触发类型（半程 / 全程 / 连击 / 通用）与年龄段分类，随机取用避免重复。年龄段由孩子 `Grade` 映射（默认：1–2 年级→低年级、3–4 年级→中年级；家长自定义语默认通用、不分龄）。

## 6. 领域模型（持久化到 MySQL）

采用 ABP DDD 分层。主要聚合根 / 实体（字段为要点，非最终定稿）：

- **ChildProfile**（聚合根）：`Id, IdentityUserId, DisplayName, Grade, AvatarKey, Pin?, ActivePetId, CreatedByParentId`。1:1 关联一个 Child 身份用户；承载该孩子的游戏档案。
- **Pet**：`Id, ChildId, Species, Name, Stage(成长阶段), GrowthPoints(单调只增·历史最高累计已得星星), CreationTime`。**不持久化每日心情/饱食**——那是从"今天的 `DailyScore`/完成度"实时推导的（§5.5），避免定时重置。v1 每个孩子一只当前宠物（结构上允许未来多只/收集）。
- **WeeklyTaskTemplateItem**：`Id, ChildId, DayOfWeek, Title, Subject, Order, EstimatedMinutes?, IsActive`。构成每周模板。
- **DailyTask**：`Id, ChildId, Date, Title, Subject, Order, SourceTemplateItemId?, IsCompleted, CompletedTime?, ReviewState(正常/被撤销)`。每天从模板惰性生成、可被家长临时改。
- **DailyScore**：`Id, ChildId, Date, TasksTotal, TasksCompleted, Stars, IsFull(吃饱), IsRestDay(N=0 无任务日)`。**记分账本 / 唯一真相源**，经"过去日补档"（§7.7）保持**无缺口**；日榜、总榜累计、家庭大目标进度、连击均由它派生（聚合 / 推导），不额外持久化可被改动的汇总值。`TasksTotal=0` 即休息日（§5.1）。
- **StreakInfo**：`ChildId, CurrentStreak, LongestStreak, LastFullDate`。仅为由 `DailyScore` 完整账本**可重算的投影 / 缓存**，非真相源（§5.3）。
- **FamilyGoal**：`Id, Title, TargetStars, RewardText, StartDate, EndDate, AchievedTime?`。进度按范围内 `DailyScore` 之和计算。
- **EncouragementMessage**：`Id, Text, TriggerType(半程/全程/连击/通用), AgeBand?(低年级/中年级), Source(内置/家长), IsActive`。
- **（可选）MilestoneEvent**：`Id, ChildId, Date, Type, Payload`。里程碑触发历史，便于展示"成就时间线"（可 v2）。
- **ABP 内置**：Identity（账号/角色）、Permission（权限）、Setting（积分/里程碑参数）、AuditLog（打卡审计）。

关系要点：`ChildProfile 1—* WeeklyTaskTemplateItem`、`ChildProfile 1—* DailyTask`、`ChildProfile 1—* DailyScore`、`ChildProfile 1—1 Pet(当前)`、`ChildProfile 1—1 StreakInfo`、`FamilyGoal` 家庭级（跨两个孩子）。

## 7. 技术架构

### 7.1 分层（ABP 标准）
- `Domain`：实体、领域服务（结算星星、宠物成长、连击、里程碑判定、休息日 / 补档处理）。
- `Application`：应用服务（今日任务、打卡、排行榜、家长后台 CRUD、家庭大目标），DTO。
- `EntityFrameworkCore`：EF Core + **MySQL** provider（实现时确认用 Pomelo 或官方 MySql provider，与 ABP 10 兼容版本对齐）。
- `HttpApi` / `Web`：MVC 单体，既托管家长后台（ABP 自动生成 UI），也托管孩子游戏端自定义页面 + 游戏端 API 端点。

### 7.2 认证与权限
- ABP Identity + OpenIddict。家长 = Admin；孩子 = Child 角色。
- 设备记住身份：首次选择后签发长效持久化登录（cookie/refresh token 绑定设备）；可选 PIN 校验。
- **PIN 生命周期**：可选的 4 位 PIN 由**家长在后台为每个孩子设置 / 重置**；孩子忘记时家长后台一键重置（孩子端不自助改 PIN）。未设置 PIN 时切换身份即直接进入。
- 权限：Child 仅能访问自身 `ChildId` 数据；排行榜只读对方公开分数字段。

### 7.3 设置项（ABP Settings）
- 星星上限、取整方式、里程碑阈值（半程%/全程/连击天数）、成长阶段阈值、日界时区等，做成设置项，家长后台可调。

### 7.4 孩子游戏端
- 服务端 MVC 页面 + 现代 CSS 动画 + 轻量 JS（宠物、喂食、庆祝动画）；通过 ABP 应用服务/接口读写数据。
- **PWA**：manifest + service worker，支持"添加到主屏幕"、离线展示今日任务外壳；打卡等写操作 v1 需在线（家庭 WiFi），离线打卡队列同步留 v2。

### 7.5 移动端优先（mobile-first）响应式 ★用户强调
- 游戏端以**竖屏手机**为首要设计尺寸，自适应放大到平板 / 桌面；触控友好（大按钮、易点区域）。
- 家长后台用 ABP 自带响应式主题，手机上亦可操作。
- 所有样机/断点以手机宽度（~360–414px）为基准起步。

### 7.6 数据与部署
- 单一 MySQL 为唯一数据源；两台设备联网访问同一后端 → 日榜/总榜天然实时一致。
- 单体应用，用户**手动部署**到服务器（一个 .NET 应用 + 一个 MySQL）。

### 7.7 日界、每日任务生成与派生状态（关键架构决策）
- **尽量少存"每日可变状态"，多用"从账本推导"**，以规避定时任务的脆弱性：
  - **每日任务生成**：**惰性生成**为准——某孩子某天首次被访问（孩子打开 / 家长查看）时，若当天还没有 `DailyTask`，就按当天星期从每周模板生成。可选加一个夜间 ABP `BackgroundWorker` 预生成（仅为体验/提醒，非正确性依赖，可 v2）。
  - **过去日的结算 / 补档（关键）**：访问时不仅处理"今天"，还会**把从上次结算日到昨天之间尚未结算的过去日逐日补齐 `DailyScore`**：该星期几模板 `N=0` → 休息日行；`N>0` 且无完成记录 → `C=0`（漏做）行。由此 `DailyScore` 成为**无缺口的完整账本**，连击 / 总榜 / 大目标据行推导才正确（解决"没人打开 App 的漏做日被连击忽略"的问题）。补档量由间隔天数决定，成本极小。
  - **宠物每日心情/饱食**：不存储，实时由"今天的完成度"推导（§5.5）——"次日满血"自动成立，无需 0 点重置作业。
  - **星星 / 连击 / 日榜 / 总榜 / 大目标进度**：由 `DailyScore` 账本聚合或推导（§4.2、§5.3）。
- **日界**：以服务器本地 0 点为"当天"边界；时区做成设置项（§7.3）。跨天只是"日期变了 → 取到新的空白当天数据"，无需迁移或重置作业。

## 8. 视觉设计（Style A · 柔和圆润 · 扁平卡通）

- **配色**：柔和马卡龙色——天蓝 `#7CB8F5`、蜜桃 `#FFB89C`、薄荷 `#8FD9B6`、薰衣草 `#C3B3F0`；白卡片，页面浅色渐变；主文字 `#2B4C6F`，次要 `#7A93AC`。
- **形状**：大圆角（16–24px）、柔和阴影、圆润字体、留白多、不过度装饰。
- **动效**：轻柔（喂食/成长/庆祝有动画但不刺眼），契合"可爱但不花哨"。
- 宠物最终采用精致插画（替换占位 emoji），风格与 Style A 统一。

## 9. 范围与分期

### v1（首个可上线版本 —— 完整核心闭环）
- 账号与登录（家长 + 两娃，设备记住身份，可选 PIN）。
- 家长后台：两娃档案、每周任务模板、当日任务临时增删改、复核/撤销打卡、家庭大目标设置。
- 孩子游戏端（Style A，移动端优先，PWA）：今日任务列表、打勾→喂宠物动画、饱食度、全做完吃饱+奖励、开局自选宠物（3 种）、宠物基础成长阶段、温和饥饿+安全护栏、里程碑鼓励（内置语库）、每日星星、休息日处理。
- 排行榜：日榜（星星）+ 总榜（累计星星 + 连击）。
- 合作大目标：进度条 + 达标庆祝 + 奖励展示。
- 持久化：MySQL；积分/里程碑参数为可调设置项。

### v2+（长期迭代）
- 更丰富的进化树 / 多宠物收集 / 装扮。
- 家长自定义鼓励语、成就徽章/时间线、进度图表/周报。
- 音效、季节/主题皮肤、提醒推送、离线打卡队列、streak 补签、夜间预生成 worker。

## 11. 成功标准

- 两个孩子每天愿意主动打开游戏、完成并打卡当天全部任务（吃饱）。
- 低龄孩子不因"总分落后"而放弃：日榜常见满星平局，弱者有当第一的体验。
- 无孩子因宠物机制产生焦虑/愧疚/逃避。
- 家长能在 5 分钟内完成"排一周任务 / 复核 / 设大目标"。
- 数据稳定持久（换设备/重装不丢，服务器一处为准）。
- 手机端（竖屏）操作顺畅、触控友好。

## 12. 待确认 / 风险

- **ABP 10 版本细节**：实现时以真实 ABP 10 模板为准核对（.NET 版本、MySQL provider 选型与兼容版本、Identity/OpenIddict 配置、模块引用），不照搬旧版记忆。
- **设备记住身份的安全度**：长效登录 vs 安全，家庭内网场景可接受；PIN 为可选增强。
- **鼓励语内容**：需产出一套高质量分龄中文语库（成长型思维口吻），并经家长审阅。
- **宠物美术**：v1 固定 **3 种**可选宠物、先用 emoji/简单插画占位；后续升级为精致插画的产出方式（自绘/生成/委托）待定（不影响 v1 逻辑）。
- **连击严格度**：默认"100% 吃饱才算连击"，若过严可放宽为"≥某完成度"，上线后据实际调。

## 13. 决策记录（头脑风暴结论）

1. 多设备、真后端（ABP 10 + MySQL），用户手动部署，长期项目。
2. 任务：每周模板惰性生成当日任务、可微调；两娃各自独立。
3. 完成：孩子自评即时反馈 + 家长事后可复核/撤销（撤销触发派生量重算）。
4. 登录：每台设备记住身份自动登录，可选 4 位 PIN（家长设/重置）。
5. 竞争基准：每日"完成度星星"，封顶 5★；日榜每天清零；`N=0` 为休息日。
6. 总榜：累计星星 + 连续打卡（连击休息日桥接、缺档漏做断连击）。
7. 合作层：两人星星合力的家庭大目标 → 解锁全家奖励。
8. 宠物情绪：温和饥饿（会蔫不会死），安全护栏（不哭闹/不生病/次日满血/进度不掉）。
9. 宠物成长：会长大进化（`GrowthPoints` 单调只增·历史最高累计星星）；开局自选 3 种之一。
10. 鼓励语：内置分龄成长型语库 + 家长可自定义。
11. 游戏端技术：ABP MVC 单体 + 自定义游戏页（单一部署单元），PWA。
12. 视觉：Style A 柔和圆润扁平卡通。
13. **移动端优先**（用户强调）：竖屏手机为主、自适应到平板/桌面。
14. **架构基调（评审修订）**：`DailyScore` 为唯一记分账本，星星/连击/日榜/总榜/大目标/宠物每日状态尽量**从账本推导**而非独立存储；访问时对过去日补档使账本**无缺口**，规避除零、级联重算、定时任务、缺档漏判四类脆弱点。
